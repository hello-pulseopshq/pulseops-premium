# PulseOps Shopify Sidekick Concept Generation Standard

**Status:** v0.1 (Derived from the Human Story chapter. Update after
additional chapters are completed.)

## Purpose

Shopify Sidekick is used only for **creative exploration**.

Its responsibility is to generate world-class editorial concepts---not
implementation, code, Liquid, CSS, schema, or engineering solutions.

Think like an Editorial Creative Director, not a Shopify developer.

------------------------------------------------------------------------

# Role

Sidekick should:

-   Explore ideas.
-   Challenge obvious layouts.
-   Generate multiple premium directions.
-   Explain *why* each concept works.

Sidekick should **not**:

-   Write code.
-   Recommend implementation details.
-   Suggest CSS/Liquid structure.
-   Optimise for engineering simplicity.

------------------------------------------------------------------------

# Required Inputs

Every prompt should include:

-   Chapter name
-   Narrative role
-   Previous chapter
-   Next chapter
-   Product category
-   Merchant goals
-   Premium benchmark brands
-   Existing PulseOps Design Languages
-   Current constraints (if any)

------------------------------------------------------------------------

# Required Output

Generate **3--5 fundamentally different concepts**.

Each concept must contain:

1.  Concept Name
2.  Narrative Purpose
3.  Memory Anchor
4.  Emotional Goal
5.  Desktop Composition
6.  Mobile Composition
7.  Merchant Benefit
8.  Cross-Vertical Adaptability

8. Cross-Vertical Adaptability

    Explain how the concept translates across the PulseOps platform without changing the underlying architecture, chapter purpose, or buyer journey.

    Describe how the same concept would adapt for at least:

    -  Supplements
    -  Skincare
    -  Coffee
    -  Electronics
    -  One additional PulseOps vertical

    Adapt through photography, copy, content, and product context—not by changing the composition or narrative role.

9.  Strengths
10.  Weaknesses
11. Why it feels premium

Avoid cosmetic variations of the same idea.

------------------------------------------------------------------------

# Design Principles

Every concept should:

-   feel editorial rather than ecommerce
-   have one dominant memory anchor
-   use intentional whitespace
-   exploit desktop space rather than stretching mobile
-   design desktop and mobile independently
-   respect the PulseOps Design Languages
-   preserve the chapter's architectural responsibility
-   scale across PulseOps verticals without changing the underlying composition

------------------------------------------------------------------------

# Quality Checklist

Reject concepts that:

-   resemble generic Shopify sections
-   rely on sliders or unnecessary carousels
-   overuse cards or equal-weight grids
-   feel template-driven
-   duplicate adjacent chapter roles
-   answer a different buyer question than the chapter owns
-   depend on vertical-specific metaphors that cannot scale across PulseOps

Prefer concepts that:

-   create a memorable scroll experience
-   tell one clear story
-   feel calm and premium
-   could become a flagship demo

------------------------------------------------------------------------

# Workflow

1.  Sidekick generates concepts.
2.  ChatGPT selects and refines one direction.
3.  ChatGPT creates blueprint.
4.  ChatGPT creates desktop and mobile mockups.
5.  Cursor implements.
6.  ChatGPT reviews.
7.  Cursor performs one refinement.
8.  Freeze.

------------------------------------------------------------------------

# Lessons from Human Story

-   Multiple concepts produced better outcomes than refining one idea.
-   Desktop and mobile must be conceived independently.
-   Mockups remove ambiguity before implementation.
-   Cursor should never invent the design.
-   Visual fidelity matters more than technical checklist completion.

This document will evolve after Ingredients, Routine, Manufacturing and
FAQ are completed.

------------------------------------------------------------------------

# Architecture Validation

Before finalizing each concept, verify that it:

-   answers only the assigned buyer question
-   does not overlap the previous chapter
-   does not overlap the next chapter
-   preserves the chapter's emotional role
-   preserves the chapter's memory anchor
-   could be reused across future PulseOps verticals
