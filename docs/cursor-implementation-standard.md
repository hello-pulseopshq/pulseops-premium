# Cursor Implementation Standard

## Mission

Implement the approved design. Do not redesign.

## Source of Truth

1.  Desktop mockup
2.  Mobile mockup
3.  Blueprint
4.  PulseOps Design Languages
5.  Existing Architecture
6.  Existing Design Tokens

## Design Languages

-   Architecture: reusable, configurable, no hardcoding.
-   Narrative: preserve chapter purpose and pacing.
-   Composition: desktop/tablet/mobile are intentional compositions;
    never stretch mobile to desktop.
-   Typography: use existing hierarchy and scales.
-   Surface: existing tokens only.
-   Photography: photography carries emotion.
-   Interaction: minimal, purposeful.
-   Motion: subtle, accessible.

## Implementation

Define desktop, tablet, mobile, merchant configurability, accessibility,
allowed files and prohibited files.

## Acceptance

-   Matches desktop mockup.
-   Matches mobile mockup.
-   Editorial rhythm preserved.
-   Desktop intentionally designed.

## Validation

Theme Check, Playwright, accessibility, regression.

## Freeze

Verify merchant configurability, runtime, then commit/tag.
