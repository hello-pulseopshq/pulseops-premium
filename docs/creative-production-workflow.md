# PulseOps Creative Production Workflow

## 1. Review Gate

Review both desktop and mobile before starting a new chapter.

### Desktop

-   Editorial story remains cohesive
-   Horizontal space used intentionally
-   No stretched mobile layouts
-   Balanced neighbouring chapters
-   Intentional whitespace

### Mobile

-   Calm rhythm
-   Readable typography
-   Strong photography
-   Smooth transitions
-   Comfortable touch targets

### Cross-device

If mobile feels stronger than desktop, identify why and improve the
desktop composition rather than copying mobile.

## 2. Creative Exploration (Sidekick)

Generate 3--5 concepts. No code.

## 3. Design Direction

Choose one concept and define narrative, memory anchor,
desktop/tablet/mobile composition.

## 4. Blueprint

Document narrative, composition, accessibility, merchant configurability
and implementation constraints.

## 5. Mockups

Approve desktop and mobile mockups.

## 6. Implementation Specification

Translate annotated mockups into structural contract per **PulseOps Implementation Specification Standard**.

## 7. Phase 1 — Composition Build

Follow the Cursor Implementation Standard. Composition fidelity only.

## 8. Composition Approval

Screenshot comparison against annotated mockups. Human visual approval required.

## 9. Phase 2 — Production Hardening

Merchant configurability, accessibility, Theme Check, implementation quality.

## 10. Review

Compare implementation to approved mockups.

## 11. Refinement

Maximum one polish sprint. Rebuild DOM if structural mismatch persists.

## 12. Freeze

Verification → Commit → Tag → Push.

For the canonical workflow, see **PulseOps Production Playbook**.
