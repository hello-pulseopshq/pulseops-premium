# Product Page Recipe

> Placeholder — canonical SP Core product page wiring guide (coming soon).

## Default stack (`product.json`)

1. **Main Product** — Dawn buy box (unchanged architecture)
2. **SP Trust Icons** — post-ATF reassurance row
3. **SP Benefits** — why customers choose this product
4. **SP Features** — alternating feature story
5. **SP Scientific Proof** — evidence / certifications
6. **SP Social Proof** — reviews
7. **SP FAQ** — objections before checkout
8. **SP CTA Offer** — closing offer
9. **SP Sticky ATC** — mobile sticky bar

## Principles

- Each section should add new information (avoid duplicate trust/shipping/guarantee copy).
- Keep ratings, review counts, and guarantees consistent within a single product story.
- Assign demo presets via **Products → Theme template** when testing verticals.
